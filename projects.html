Automated E2E Testing Utility

Visa inc. Internship 2016

In the summer of 2016 I worked for Visa inc. at their headquarteres in Foster City CA. Inefficiency in testing lead me to write an automation script to streamline the E2E testing process. The script was wrritten in Java using Solenium and OpenSSL for authentication of API calls. The utility was implemented in a genereic manner so that it could test multiple internal sites to check the full end 2 end flow. The script is easily expandable to more apis that might appear on the VDP site. The automation generates a html health report of the particular website. It is currently implemented in the CICD pipeline.

The Search for an Economic Sensor

VADER Lab Lehigh

Robotic perception and localization accuracy are functions of sensor quality. The error in standard odometry information grows exponentially. In the robot world sensors are very important and unfortunatley even a "good enough" stereo camera is hard to come by for a reasonable price. I started work at VADER lab in 2014. I worked on a variety of projects including website upkeep and most recently sensor testing. Since the Kinect was puchased by Apple the robotics community had trouble finding a low cost camera to use for stereo vision. After creating and running a sereies of validation checks including error calculatons, I found many options but settled on one sensor traditionally used for biometric retna scans for security purposes. This work heavily relyed on the use of Matlab.
